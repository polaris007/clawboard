# 新增API接口文档

## 模块四：系统管理接口

本模块用于系统管理和维护操作。

### 8. 清空数据库所有表

清空数据库中所有表的数据。此操作不可恢复，请谨慎使用。

* **路径**: `/api/v1/admin/reset-database`
* **方法**: `POST`
* **请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| confirmCode | String | 是 | 确认码，格式为当天日期的YYYYMMDD格式，如"20260421" |

* **成功响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": "数据库已成功清空"
}
```

* **错误响应示例（确认码无效）**:

```json
{
  "code": 500,
  "message": "确认码无效或已过期，请使用当天的日期作为确认码",
  "data": null
}
```

## 使用示例

### 测试错误的确认码
```bash
curl -X POST "http://localhost:8080/api/v1/admin/reset-database?confirmCode=20260420"
```

### 测试正确的确认码（假设当天是2026-04-21）
```bash
curl -X POST "http://localhost:8080/api/v1/admin/reset-database?confirmCode=20260421"
```
